# Codebase conventions

Distilled from `CLAUDE.md` at the repo root. If this file and `CLAUDE.md` ever drift, `CLAUDE.md` wins — update this one.

## Per-service package layout

Each new resource/data source lives in `internal/services/<service>/` with this file structure:

| File | Purpose |
| --- | --- |
| `<name>_model.go` | Terraform state/plan struct with `tfsdk` tags. |
| `<name>_helper.go` | API ↔ model conversion helpers. Optional — fold into `_model.go` if small. |
| `<name>_resource.go` | `resource.Resource` impl: schema, CRUD, timeouts, plan modifiers. |
| `<name>_data_source.go` | `datasource.DataSource` impl. |
| `<name>_resource_test.go` | Unit + acceptance tests for the resource. |
| `<name>_data_source_test.go` | Unit + acceptance tests for the data source. |
| `acc_test.go` | Shared acceptance test scaffolding for the package. |

Other touch points:

- `internal/provider/provider.go` — register the new `Resources` / `DataSources` factory. **Forgetting this leaves the type invisible even though tests compile.**
- `examples/<service>/main.tf` — runnable example referenced by docs.
- `website/docs/r/<name>.html.markdown` and `website/docs/d/<name>.html.markdown` — published docs (legacy layout).

## Schema rules

- **Every attribute needs a `MarkdownDescription`.** It flows into the generated schema docs. No exceptions.
- **`UseStateForUnknown`** on every computed ID/timestamp. Missing this causes spurious diffs.
- **`RequiresReplace`** on every immutable field.
- **Data sources look up by `id`** (see `nscale_ssh_certificate_authority` for the canonical pattern). Don't expose name-based lookup unless the API supports it directly.
- **Timeouts** via `timeouts.Attributes{Create: true, Update: true, Delete: true}` (drop `Update` for immutable resources).

## Docs

The repo uses the **legacy `website/docs/{r,d}/*.html.markdown`** layout, not the newer Registry `docs/` layout.

1. Run `make generate` (or `cd tools && go generate ./...`). This invokes `tfplugindocs` and writes output to `./docs/` at the repo root.
2. **Discard the `docs/` directory** — not tracked, not the pattern this repo uses.
3. Copy the generated schema block into hand-authored files under `website/docs/r/` and `website/docs/d/`, adapting the front matter + heading to the repo's style:
   - Front matter: `page_title: "Nscale: nscale_<name>"`, `subcategory: ""`, `description: Nscale <Title Case>`.
   - Heading: `# Resource: nscale_<name>` or `# Data Source: nscale_<name>` (not the generator's `# nscale_<name> (Resource)`).
   - Write a short prose description and an `## Example Usage` block by hand (the generator leaves these empty because there are no templates under `templates/`).
   - Paste the `<!-- schema generated by tfplugindocs -->` section unchanged.

## Verification

```sh
make fmt lint       # gofmt + golangci-lint
make test           # unit tests
make testacc        # acceptance tests (requires live API credentials; gated behind TF_ACC=1)
```

Confirm the example in `examples/<service>/main.tf` actually applies against a real project before shipping.

## Don't

- **Don't modify `tools/tools.go`** beyond what came from the Terraform provider scaffolding — verbatim copy. The `//go:build generate` tag sits **below** the Apache license header.
- **Don't commit state files** from `examples/<service>/` (`terraform.tfstate*` are local-only).
- **Don't commit the generated `docs/` directory** — only `website/docs/` is tracked.
